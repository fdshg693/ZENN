## はじめに

AIを効果的に活用するためには、適切なプロンプト（指示文）を書くことが重要です。この記事では、プロンプトの基本原則と、避けるべきアンチパターンを紹介します。

## 基本原則

### 1. 明確で具体的に書く
- 曖昧な表現を避け、具体的な指示を出す
- 期待する出力形式を明示する
- 必要な情報をすべて含める

### 2. 構造化する
- 箇条書きや番号付きリストを活用
- 複雑なタスクは段階的に分解
- 役割・コンテキスト・タスク・出力形式の順で整理

### 3. 例を示す（Few-shot prompting）
- 入力と期待される出力の例を1〜3個提示
- AIが正しいパターンを学習しやすくなる

## 🚫 避けるべきアンチパターン

### ❌ アンチパターン1: 曖昧すぎる指示

**悪い例:**
```
いい感じの文章を書いて
```

**問題点:** 「いい感じ」が何を意味するのか不明。長さ、トーン、目的が分からない。

**良い例:**
```
20代女性向けのカジュアルなトーンで、
新商品のスニーカーを紹介する200文字程度のSNS投稿文を書いてください。
商品の特徴: 軽量、カラフル、低価格
```

---

### ❌ アンチパターン2: 複数タスクを一度に詰め込む

**悪い例:**
```
このコードをレビューして、バグを直して、テストを書いて、ドキュメントも作って
```

**問題点:** タスクが多すぎて、各タスクの品質が低下する。優先順位も不明。

**良い例:**
```
以下のコードについて、まずバグの有無をチェックしてください。
バグがあれば指摘と修正案を提示してください。

[コード]

※テストとドキュメントは次のステップで依頼します
```

---

### ❌ アンチパターン3: 出力形式を指定しない

**悪い例:**
```
Python、JavaScript、Goの違いを教えて
```

**問題点:** 長文の説明なのか、箇条書きなのか、表形式なのか分からない。

**良い例:**
```
Python、JavaScript、Goの違いを以下の観点で比較し、
Markdown形式の表にまとめてください。

比較観点:
- 主な用途
- 実行速度
- 学習難易度
- 型システム
```

---

### ❌ アンチパターン4: 否定形だけで指示する

**悪い例:**
```
難しい言葉を使わないで
専門用語を避けて
長くしないで
```

**問題点:** 何を「する」べきかが不明。禁止事項だけでは方向性が定まらない。

**良い例:**
```
中学生でも理解できる言葉で説明してください。
専門用語を使う場合は、必ず簡単な言い換えを（ ）内に付けてください。
1つの段落は3文以内にしてください。
```

---

### ❌ アンチパターン5: コンテキストを省略する

**悪い例:**
```
これを直して
```

**問題点:** 「これ」が何か、どう「直す」のかが不明。

**良い例:**
```
以下のPythonコードで「TypeError: 'NoneType' object is not iterable」
エラーが発生しています。

【コード】
def get_items():
    return None

for item in get_items():
    print(item)

【期待する動作】
itemsが空の場合はエラーを出さずに処理をスキップしたい

修正案を提示してください。
```

---

### ❌ アンチパターン6: 矛盾した指示を出す

**悪い例:**
```
できるだけ短く、でも詳しく説明して
カジュアルに、でもプロフェッショナルに書いて
```

**問題点:** 相反する要求でAIが混乱する。

**良い例:**
```
3〜5文で要点を押さえて説明してください。
トーンはビジネスカジュアル（丁寧だが堅すぎない）でお願いします。
```

## ✅ 良いプロンプトのテンプレート

```
【役割】あなたは〇〇の専門家です。

【背景】△△という状況で、□□が必要です。

【タスク】以下を行ってください：
1. ○○を分析する
2. △△を提案する

【制約条件】
- 日本語で回答
- 箇条書きで整理
- 200文字以内

【出力形式】
Markdown形式で、見出しを付けて構造化してください
```

## まとめ

| ポイント | 悪い例 | 良い例 |
|---------|--------|--------|
| 具体性 | 「いい感じに」 | 「200文字、カジュアルなトーン」 |
| タスク分割 | 全部一度に依頼 | 1つずつ順番に依頼 |
| 出力形式 | 指定なし | 「表形式で」「箇条書きで」 |
| 肯定形 | 「〜しないで」 | 「〜してください」 |
| コンテキスト | 「これを直して」 | 状況・コード・期待動作を明示 |
| 一貫性 | 矛盾した要求 | 明確な優先順位 |

## おわりに

良いプロンプトを書くコツは、**AIを「人間のチームメンバー」として扱うこと**です。曖昧な指示では人間も困るように、AIも同様です。

ぜひこれらのポイントを意識して、より効果的なAI活用を実践してみてください！
