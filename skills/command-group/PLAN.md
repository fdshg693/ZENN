# コマンドグループ

## 目的

複数のSKILLを束ねたスキルを作成

- 複数のSKILLをまとめて、AIに読み込ませる

## `skill-group`スキルとの比較

1. スキルの使い方

- `skill-group`スキルはAIからの読み込みも想定しており、様々なスキルを探索して利用することを想定している。
- `command-group`スキルはAIからの読み込みは行わず、ユーザーが必要なスキルを正確に指定して呼び出すことを想定している。

2. スキルのパフォーマンス

- `skill-group`スキルは、AIがスキルを探せる柔軟性があるが、スキル名列挙->スキル概要取得->スキル内容取得のステップを踏むため、スキル利用までのステップが多い。
  - スキル名列挙は最初だけで済むが、その後の2ステップはスキルごとに必要になる。
- `command-group`スキルは、ユーザーが必要なスキルを正確に指定して、スキルの内容をまとめてAIに渡せる
  - 複数のスキルを利用する場合であっても、1回のスキル呼び出しで済むため効率的

## ユースケース

- AIから使われることはない
- 同階層にBashコマンドを作成
  - 引数を受け取って上のような挙動を行うスクリプトにする
    - SKILL側でそれを !`command args` の形で埋め込む
    - SKILLでは $ARGUMENTS の形で引数を参照できる機能を利用して引数をシェルスクリプトに渡す
  - SKILLは `.claude/skills/command-group/sub_commands`配下の内容から動的に取得
    - `SKILL.md`を再帰的に探して、フロントマターのnameが一致するものを見つける
    - 見つからなかった場合はスキップ
    - 見つかったSKILLの内容をまとめて埋め込む（各SKILLごとのパスも記述することで、SKILLに記載された同階層のファイルへの参照などが意味をなすようにする）

- ユーザー
  - /command-group skill1 skill2 のようにコマンドを呼び出して、AIに直接必要なスキルを提供
